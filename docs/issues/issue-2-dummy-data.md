# Issue #2: 型定義・ダミーデータ・モックAIの実装 (feature/dummy-data)

## 📌 概要
アプリ全体で使用するTypeScriptの型定義、教員や学生の初期プロフィール・相談テーマなどを定義したダミーデータ、および7つのモックAI関数を実装します。

## 🎯 ゴール
- アプリで必要なデータ構造（`UserProfile`, `ConsultRequest`, `TimeSlotScore`, `MailOutput`など）が正しく定義されていること
- 教員・学生・職員を含む、バリエーション豊かなダミーデータが用意されていること
- 7つのAI処理モジュール（入力汲み取り、相談先選定、プロフィール解析、日程スコアリング、プライバシー調整、メール生成、メールチェック）が、入力に対して適切な動作（ルールベースの疑似AI挙動）を示すこと

## 🛠️ 作業手順
1. `src/types/index.ts` に型定義を作成する
2. `src/lib/dummyData.ts` に教員のプロフィール、対応可能トピック、メール方針などのダミーデータを作成する
3. `src/lib/ai/` ディレクトリを作成し、以下のモジュールを作成する
   - `parser.ts` (自由文から相談件名、所要時間、急ぎ度を抽出)
   - `selector.ts` (リクエストと教員のtopicsから相談先をスコアリング)
   - `analyzer.ts` (プロフィールの自由文から条件を抽出)
   - `scorer.ts` (自分と相手の空き時間を重ね合わせ、スコア◎○△を決定)
   - `privacy.ts` (予定をぼかした表現に変換)
   - `mailGen.ts` (メールの自動生成)
   - `mailCheck.ts` (学籍番号有無や失礼度の検証)
4. 動作テスト用の簡易スクリプトを `scratch/` で実行するか、直接インポートできるようにする
5. 変更をコミットし、`main` にマージする
